(AWS EC2) Service for EC2 指定固定IP及釋放IP

前言

當你開啟一個EC2 Instance
會給你一個非固定IP，自己練習或許不需要
(AWS EC2 -> 選擇EC2 -> 左上角Action -> Instance State -> Reboot <<<做此動作因為非固定IP，等重開完畢IP 會變，但使用terminal連進EC2 Instance 下命令sudo reboot重開機則IP不會變，因為一個是AWS主控台機器重開和你連進Instance重開是不一樣的，主控台會依據你有無設定固定IP去做綁定)

如果專案需要，那固定IP設定方式如以下步驟
(固定IP指定給EC2 Instance，使用中不會收費，如果你stop EC2 Instance且沒釋放掉IP，那麼你的信用卡帳單就會多出這一筆，請注意)

登入AWS

選擇區域

選擇Service

選擇NETWORK & SECURITY -> Elastic IPs

選擇Allocate New Address

彈出一個視窗，問你是否要一個IP
Yes, Allocate

固定IP申請成功

固定IP指定給EC2 Instance

點擊申請的固定IP -> 選擇左上角Actions -> Associate Address

彈出設定頁面


Instance:選擇要指定的EC2 Instance

固定IP指定成功

測試是否指定成功

INSTANCES -> Instances

重要！！！

IP不使用記得釋放掉
步驟如下

選擇左上角Actions -> Disassociate Address

清除EC2之間的關聯

Yes, Disassociate

清除關聯成功

釋放IP

選擇左上角Actions -> Release Address

Release Address

Yes, Release，即釋放此IP

原本IP要指定到另一台EC2 Instance

點擊IP -> 選擇左上角Actions -> Disassociate Address -> 點擊IP -> 擇左上角Actions -> Associate Address -> 彈出設定頁面 -> 選擇要指定的EC2 Instance -> 固定IP指定成功
